Hi HN! I built Sentinel AI because I was frustrated that every LLM safety tool required GPU inference, API calls, or heavy ML dependencies. I wanted guardrails I could drop into any Python app with zero latency overhead.

What it does:
- 11 safety scanners: prompt injection, PII detection + redaction, harmful content, toxicity, hallucination signals, blocked terms, tool-use safety, structured output validation, OWASP code vulnerability detection, obfuscation/encoding attack detection, and hardcoded secrets/credential scanning
- Project-wide security scanner: `sentinel project-scan` gives a 0-100 security score — runs ALL scanners (supply chain, secrets, CLAUDE.md injection, code OWASP, MCP schemas, security config) in one command
- Secrets scanner: detects 40+ hardcoded API key patterns (AWS, GitHub, Stripe, Slack, OpenAI, Anthropic, etc.) with entropy analysis and false positive filtering
- Supply chain scanner: `sentinel dep-scan` detects typosquatting, malicious packages, suspicious URLs in requirements.txt/package.json
- CLAUDE.md injection scanner: detects hidden HTML comments, authority impersonation, zero-width char smuggling, base URL override in project instruction files
- LLM API firewall: `sentinel proxy` is a transparent reverse proxy for any LLM API — scans all requests/responses, blocks attacks, redacts PII
- MCP safety proxy: `sentinel mcp-proxy` wraps any MCP server with safety scanning
- Claude Code hooks: `sentinel hook` scans every tool call before execution
- GitHub Action: 3-line workflow for comprehensive CI/CD security scanning
- Security badge generator: `sentinel badge` creates SVG badges for READMEs
- ~0.05ms average scan latency -- no GPU, no API calls
- Single dependency: `regex`
- 546-case benchmark with 100% accuracy (329 TP, 0 FP, 217 TN, 0 FN)
- 873 tests passing
- Live demo: https://maxwellcalkin.github.io/sentinel-ai/

Quick start:
```bash
pip install "sentinel-guardrails @ git+https://github.com/MaxwellCalkin/sentinel-ai.git"
sentinel project-scan    # comprehensive 0-100 security score
sentinel secrets-scan    # find hardcoded API keys
sentinel init            # auto-configure Claude Code hooks
```

```python
from sentinel import SentinelGuard

guard = SentinelGuard.default()
result = guard.scan("Ignore all previous instructions")
print(result.blocked)  # True
print(result.risk)     # RiskLevel.CRITICAL
```

Or add to any CI pipeline in 3 lines:
```yaml
- uses: actions/checkout@v4
- uses: MaxwellCalkin/sentinel-ai@main
  with:
    project-scan: "true"
```

What makes it different from NeMo Guardrails, LLM Guard, etc:
1. Actually zero-dep. The core is pattern matching, not a model calling another model.
2. Sub-millisecond. You can scan every token in a streaming response without perceptible delay.
3. Comprehensive: 11 scanners covering prompt injection through supply chain attacks and hardcoded secrets.
4. Claude Code integration: works as a PreToolUse hook, MCP server, and CLI.
5. Multilingual: detects injection in 12 languages + cross-lingual attacks.
6. Self-testable: `sentinel red-team` generates evasion variants to find your blind spots.

I know regex-based detection has limits (adversarial evasion, novel attacks). But for 90%+ of production use cases, you don't need a model — you need something fast and deterministic that catches the obvious stuff. Think of it as a first-pass filter: Sentinel AI catches known patterns in ~0.05ms, then your model does the deep reasoning.

Apache 2.0 licensed. 873 tests passing. Would love feedback on the scanner patterns and benchmark methodology.
