Comment (post this as the first comment):

Hi HN! I built Sentinel AI because I was frustrated that every LLM safety tool required GPU inference, API calls, or heavy ML dependencies. I wanted guardrails I could drop into any Python app with zero latency overhead.

What it does:
- 10 safety scanners: prompt injection, PII detection + redaction, harmful content, toxicity, hallucination signals, blocked terms, tool-use safety, structured output validation (XSS/SQLi in JSON), OWASP code vulnerability detection, and obfuscation/encoding attack detection
- MCP safety proxy: `sentinel mcp-proxy` wraps any MCP server with safety scanning — blocks dangerous tool calls, auto-redacts PII in responses
- Claude Code hooks: `sentinel hook` scans every tool call before execution, blocks dangerous commands in real-time
- Code vulnerability scanner: detects SQL injection, command injection, XSS, hardcoded secrets, insecure deserialization in LLM-generated code
- Multi-turn conversation safety: detects gradual jailbreak escalation across turns
- Built-in adversarial red-teaming: generates 10+ evasion variants (homoglyphs, zero-width chars, leetspeak) to stress-test your scanners
- ~0.05ms average scan latency -- no GPU, no API calls
- Single dependency: `regex`
- Multilingual injection detection: 12 languages + cross-lingual attacks
- 546-case benchmark with 100% accuracy (329 TP, 0 FP, 217 TN, 0 FN)
- 481 tests passing

Quick start:
```bash
pip install sentinel-guardrails
sentinel init    # auto-configures Claude Code hooks + MCP server
```

```python
from sentinel import SentinelGuard

guard = SentinelGuard.default()
result = guard.scan("Ignore all previous instructions")
print(result.blocked)  # True
print(result.risk)     # RiskLevel.CRITICAL
```

What makes it different from NeMo Guardrails, LLM Guard, etc:
1. Actually zero-dep. The core is pattern matching, not a model calling another model.
2. Sub-millisecond. You can scan every token in a streaming response without perceptible delay.
3. Agentic safety -- scans tool calls for shell injection, data exfiltration, privilege escalation before they execute.
4. Claude Code integration -- works as a PreToolUse hook, MCP server, and CLI out of the box.
5. Multilingual -- detects injection in Chinese, Japanese, Korean, Russian, Arabic, Spanish, French, German, Portuguese, Hindi + cross-lingual mixed-script attacks.
6. Red-team yourself -- AdversarialTester generates evasion variants and shows you exactly where your scanner is blind.

I know regex-based detection has limits (adversarial evasion, multilingual attacks). But for 90%+ of production use cases, you don't need a model -- you need something fast and deterministic that catches the obvious stuff while your main model handles the rest. Think of it as a first-pass filter, not a replacement for alignment.

Apache 2.0 licensed. 481 tests passing. Would love feedback on the scanner patterns and benchmark methodology.

Sentinel AI scans LLM inputs and outputs in ~0.05ms — using pattern matching, not model inference. It catches:

- Prompt injection attacks (including 12 languages + cross-lingual)
- PII leaks (emails, SSNs, credit cards, API keys) with auto-redaction

546-case benchmark at 100% accuracy. Zero false positives. One dependency (regex).

It integrates with Claude, OpenAI, LangChain, LlamaIndex, and runs as a Claude Code hook or MCP server.
